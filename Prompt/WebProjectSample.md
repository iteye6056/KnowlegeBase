一、项目需求整理（Easy Time-off）
1. 项目目标
构建一个 统一的员工休假可视化平台（Easy Time-off），整合多来源的休假数据，支持个人、团队、组织层级的休假查询与共享，提升团队协作与资源安排效率。

2. 用户与权限
2.1 用户身份
* 公司内部员工
* 使用 Active Directory（AD）账号登录
* 支持：
    * 自动登录（Remember Me）
    * 会话保持与安全退出
2.2 权限规则
* 所有员工可以：
    * 查看 自己 的休假信息
    * 查看 任何其他员工 的休假信息（只读）
* 员工只能：
    * 新增 / 修改 / 删除自己的休假记录
    * 不能修改他人的休假数据

3. 数据来源与整合
3.1 休假数据来源
系统需整合以下三类数据：
1. HR 系统
    * 员工正式请假记录，不区分假期类型。
    * 作为权威数据源之一
2. Outlook Out of Office
    * 自动同步员工 OOO 状态
    * 用于补充非正式或临时离岗信息
3. 员工手动维护
    * 员工可手动创建、修改、删除自己的休假记录
    * 适用于特殊情况或补充说明
⚠️ 系统需明确标识数据来源（HR / Outlook / Manual）
4. Public holiday，从互联网搜索并导入2026年的public holiday，允许配置，比如中国的调休，我们公司除了春节之前不调休，但是春节调休。

4. 功能需求
4.1 个人休假管理
* 查看自己的休假列表（列表 / 日历视图）
* 新增休假，作为easy版本，用户的操作要非常简单，只需要点击开始日期，结束日志，提交就行，不需要假期的类型，备注是可选的。
* 修改休假
* 删除休假

4.2 员工休假查询
* 根据员工姓名 / ID 查询
* 查看任意员工的休假日历（只读）

4.3 组织视角日历
4.3.1 按 Agile Pod Team 查看
* Pod Team 信息来自 Teambook 系统
* 展示 Pod 内成员的休假日历
4.3.2 虚拟团队（Virtual Team）
* 用户可创建自定义虚拟团队：
    * 手动选择员工
    * 命名虚拟团队
* 查看该虚拟团队的休假日历
* 可编辑 / 删除虚拟团队

4.4 日历与分享
* 对于多人的日历视图，每个人是一行，列是日期，连续的日期，中间的cell是public holiday或者个人holiday，对于桌面版，鼠标放上去的时候高亮显示改行/列，方便超大视图定位人和日期。
* 每个日历可生成：
    * 每个视图都有可分享的访问链接（只读）
    * 用于跨团队或管理层查看
每个浏览过的视图，比如某个人的，某个Pod Team的，某个Virtual Team的视图，应该有个浏览历史，方便用户快速回看，另外加入收藏功能，让用户收藏某些团队的视图。

5. 非功能性需求
5.1 安全
* AD 统一身份认证
* API 权限校验
* 数据访问控制
* Session / Token 安全管理
5.2 可用性
* 现代、友好、直观的 UI
* 支持桌面浏览器优先（Responsive），考虑34寸带鱼屏显示器和手机的显示。
5.3 可扩展性
* 支持更多组织结构维度
